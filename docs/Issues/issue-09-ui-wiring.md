# Issue: 生成・クリア・書き出しのUI配線を実装する

## 概要
UI操作からサンプル生成・クリア・PNG書き出しを実行できるようにする。

## 目的（完了条件）
- 生成ボタンでサンプルが描画される。
- クリアボタンでキャンバスが初期化される。
- 素材用（透過・ラベル無し）PNGが保存される。
- 確認用（白背景・ラベル有り）PNGが保存される。

## ロードマップ小項目（割り当て）
- 素材用（透過・ラベル無し）/確認用（白背景・ラベル有り）の書き出しを選べるようにする（ボタン分割またはトグル等）
- 最初は `FileSavePicker` でユーザーに保存場所を選ばせる（PicturesLibrary権限依存を避ける）
- 保存形式は PNG 固定にする
- （任意）export解像度（例：4096/8192）のUIを追加する
- （任意）stroke間隔、線の長さ、pressure固定値のUIを追加する
- （任意）背景（透明/白）切替のUIを追加する

## 作業内容
- `MainPage.xaml` のボタンと `MainPage.xaml.cs` の処理を接続する。
- 生成前/生成後の状態（ボタン有効/無効など）が破綻しないようにする（必要最低限）。

## 影響範囲
- `MainPage.xaml`
- `MainPage.xaml.cs`
